# Clify

> **Clify makes your software cliable.**

Generate fully-featured CLI tools from API specifications. Define your API commands declaratively in YAML, and Clify generates a compiled Rust CLI binary with auth, output formatting, pagination, shell completions, and more.

## Installation

### Pre-built binaries (fastest)

Download the latest binary for your platform from [GitHub Releases](https://github.com/mdude/clify/releases/latest):

| Platform | Binary |
|----------|--------|
| macOS (Apple Silicon) | `clify-macos-aarch64` |
| macOS (Intel) | `clify-macos-x86_64` |
| Linux (x86_64) | `clify-linux-x86_64` |
| Linux (aarch64) | `clify-linux-aarch64` |

```bash
# Example: macOS Apple Silicon
curl -L -o clify https://github.com/mdude/clify/releases/latest/download/clify-macos-aarch64
chmod +x clify
sudo mv clify /usr/local/bin/

clify --version
```

> **Note:** Generated CLIs still need Rust to compile. Clify itself is just the generator — it produces a Rust project that you then `cargo build`.

### From source

#### Prerequisites

**Rust** (1.75+) and **Cargo** are required.

```bash
# Check if Rust is installed
rustc --version   # expect: rustc 1.75.0 or higher

# Check if Cargo is installed
cargo --version   # expect: cargo 1.75.0 or higher
```

If either is missing, install both with [rustup](https://rustup.rs/):

```bash
curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh
source "$HOME/.cargo/env"
```

#### Install

```bash
git clone https://github.com/mdude/clify.git
cd clify
cargo install --path crates/clify-cli
```

This installs the `clify` binary to `~/.cargo/bin/` (make sure it's in your `PATH`).

#### Build without installing

```bash
git clone https://github.com/mdude/clify.git
cd clify
cargo build --release
```

The binary will be at `target/release/clify`.

### Verify

```bash
clify --version
clify --help
```

## Quick Start

```bash
# 1. Scan an OpenAPI spec to generate a .clify.yaml
clify scan --from openapi ./api-spec.yaml

# 2. Review and curate the generated spec
vim api.clify.yaml

# 3. Generate the CLI project
clify generate api.clify.yaml --output ./my-cli

# 4. Build
cd my-cli
cargo build --release

# 5. Use it
./target/release/my-api --help
./target/release/my-api auth login
./target/release/my-api some-command --param value
```

Or just run `clify` with no arguments for the interactive TUI.

## Commands

| Command | Description |
|---------|-------------|
| `clify` | Launch interactive TUI |
| `clify init` | Create a starter `.clify.yaml` |
| `clify scan` | Generate spec from OpenAPI/Swagger |
| `clify validate` | Check a spec for errors |
| `clify generate` | Create a Rust CLI project from spec |
| `clify build` | Compile the generated project |
| `clify schema` | Export JSON Schema for IDE support |

## Features (v0.1)

- 📄 **Declarative specs** — YAML-based, human-readable, auto-generatable
- 🔍 **API scanning** — auto-generate specs from OpenAPI 3.x and Swagger 2.0
- ⚙️ **Code generation** — produces compilable Rust CLI projects
- 🔐 **Auth** — API key, bearer token, basic, OAuth2 with custom hooks for non-standard APIs
- 📊 **Output** — JSON (pretty/compact), table, CSV
- 📄 **Pagination** — offset, cursor, link — auto-followed
- 🐚 **Shell completions** — bash, zsh, fish, PowerShell
- 🖥️ **Interactive TUI** — guided workflows via ratatui
- 🏗️ **Single binary** — generated CLIs compile to one static binary
- ✅ **Validation** — 20+ structural checks with clear error messages

## Spec Format

See [CLIFY-SPEC.md](docs/CLIFY-SPEC.md) for the full specification.

### Minimal example

```yaml
meta:
  name: my-api
  version: "0.1.0"
  description: "CLI for My API"

transport:
  type: rest
  base_url: "https://api.example.com/v1"

auth:
  type: api-key
  location: header
  name: "Authorization"
  env: MY_API_KEY

commands:
  - name: list-users
    description: "List all users"
    request:
      method: GET
      path: "/users"
    response:
      success_path: "data"
```

## Architecture

```
clify-core      Spec parsing, validation, code generation, scanning
clify-cli       The clify binary (TUI + commands)
clify-runtime   Shared runtime for generated CLIs (auth, HTTP, output)
```

## Generated CLI Features

Every CLI generated by Clify automatically includes:

- `auth login / status / logout` — credential management
- `config set / get / list / reset` — persistent configuration
- `--output json|table|csv` — output format on every command
- `--dry-run` — preview HTTP requests without sending
- `--verbose` — show request/response details
- `--base-url` — override the default server URL
- `--token` — override auth for a single request
